17.0 - 12 February 2026

Features

  • Modern GUI: Added pages to manage the new policy engine (policy groups, rules, endpoint profiles). On the netdevice details page, added an option to assign a netdevice or any of its interfaces to a policy group. (#290771)
  • Policy: Added new, simplified policy engine for RADIUS authentication requests. (#290635)

Enhancements

  • Logging: Client certificate subject and issuer common name are now stored in connection events when performing EAP authentication using the internal RADIUS server. This information is also forwarded to syslog in the new fields certcn and certissuercn of CONNECTION_EVENT (1180) messages. (#290635)
  • Logging: WLAN SSID and access point MAC address are now stored in connection events. This information is also forwarded to syslog in the new fields ssid and apmacaddress of CONNECTION_EVENT (1180) messages. (#290831)
  • Logging: When using the internal RADIUS server and a client is rejected because of an invalid certificate, the reason as well as the certificate common name are now included in the related connection block event. The parameter exceptiontype in the syslog message will contain an appropriate exception identifier. Refer to the USP NAS user manual for a complete list of possible exceptions. (#291811)
  • Modern GUI: Add option to delete all unused SNMP access profiles (#291862)
  • Modern GUI: Added option to configure EAP-Username patterns for auto-assigning asset type and asset classes to inventory entries (#256421)
  • Modern GUI: Added option to configure SSH transfer user on application settings page (#291641)
  • Modern GUI: Added option to configure the local SNMP service on the system settings page. This includes a new option to enable/disable the local SNMPD service. On new installations, it is disabled by default. (#291642)
  • Modern GUI: Added option to define if and when non-inventoried endpoint records should be cleaned up. This can be configured on the application settings page. A new log event 1936 “Automatic cleanup of inactive nodes complete.” has been added, which is emitted whenever endpoint records are cleaned up. (#291858)
  • Modern GUI: Added option to define labels for each VLAN used in policies or observed on netdevices (#289906)
  • Modern GUI: Added page for creating and scheduling of PDF, CSV and HTML reports related to the operation, configuration and inventory of USP NAS. (#291366)
  • Modern GUI: Added page for defining the high availability (master-authenticator) configuration and synchronization (#291372)
  • Modern GUI: Added possibility to create a system dump file for support purposes on the maintenance page (#228593)
  • Modern GUI: Added system information section to maintenance page (#286314)
  • Modern GUI: Endpoints which are currently active are now marked with a small status indicator in the endpoint overview table, and the tooltip indicates the current status type. (#291931)
  • REST API: Added policyGroupName property to Netdevice, Interface and Endpoint REST-API responses, to indicate the relevant policy group linked to each record. (#291765)

Changes

  • Core: Removed obsolete SNMP trap forwarder feature (#291664)
  • Core: Removed support for soft-deleted netdevices and netports to simplify data structure and code base. The column deleted have been removed from the netdevice and netport database tables. The “Trashed netdevices” page has been removed from the Web GUI. CAUTION If you use any custom SQL in scripts, verify that these columns are not used anymore. (#291668)
  • Core: Upgraded Bouncy Castle open-source cryptographic APIs to version 1.83 to address CVE–2025–8916, a vulnerability with medium severity. (#291885)
  • Logging: CAUTION When a client performing EAP authentication is rejected by the server (e.g. invalid certificate), the related connection event is now a CONNECTION_BLOCKING_EVENT of type RADIUS_SERVER_REJECT instead of a CONNECTION_EVENT with type POLICY_HANDLER_ABORT. If log messages are forwarded via syslog and evaluated on a thirty-party log collector, some changes might be needed when using dedicated queries to analyze this data. (#292081)
  • Modern GUI: A netdevice scan for multiple devices can now be triggered by selecting the desired entries from the netdevice table, and then clicking the scan button in the bulk action bar. The scan button at the top of the page has been removed. (#291822)
  • Modern GUI: Consolidated port and netport tabs on the netdevice details page. A new “Edit” button on top of the interface table enters edit mode, where netports and other interface settings can be changed. (#291792)
  • Modern GUI: Removed option to set RADIUS flag setting in interface table on the netdevice details page. This flag is set automatically on every switch scan. Resetting the netdevice using the related button will now also reset the RADIUS flag before the next scan. (#291959)
  • Modern GUI: The login page of the modern GUI is now shown by default (#291666)
  • Modern GUI: The pages “SNMP access profiles” and “RADIUS subnets” are now part of a new “Netdevice settings” page, along with the new “VLAN mapping” feature. (#292072)
  • Modern GUI: Updated Apache POI library (the Java API for Microsoft Documents) to address a potential vulnerability classified as medium (CVE–2025–31672) (#291873)
  • Modern GUI: Updated Apache Tika content detection library to address a vulnerability reported as critical (CVE–2025–66516) (#291863)
  • RADIUS: The setting “Enable autorisation by eap-username” / “Automatically add endpoints to inventory if certificate is valid” is now on by default for new installations. (#292108)

Bugfixes

  • Logging: Fixed invalid arguments in event “Netport will be overwritten by new source” (1532). The event will now contain the device name, interface index and interface name as proper key=value arguments. (#291544)
  • Modern GUI: Enforce login after Web GUI restarts, in order to fetch new user permissions after system upgrade. (#292102)
  • Monitoring: CAUTION Due to an underlying change in how the operating systems represents information about the system memory, some values which can be fetched via SNMP for monitoring had their purpose changed. The values returned did not match anymore what their label said. The USP Appliance MIB has been adapted as follows to fix this: .1.3.6.1.4.1.26458.7419.11.3.5.0 is now labeled memoryBufferedCached instead of memoryBuffered, and OID .1.3.6.1.4.1.26458.7419.11.3.6.0 is now labeled memoryAvailable instead of memoryCached. (#291766)
  • RADIUS: When a RADIUS MAC authentication request is rejected, the log event 1153 “Radius enabled endpoint was refused to access the network (reject).” is now being emitted. Previously, this was only done for EAP authentication requests. Therefore, the report “Blocked endpoints” will now also include rejected enpoints which used RADIUS MAC authentication. (#291925)
  • Reporting: Allow to request up to 999’999 max records in the ‘Endpoint’ report and fix input validation and error messages for max records parameters in other reports (#291932)
  • Reporting: Fixed broken SQL query in report “Blocked endpoints” (#291627)
  • SAB: Fixed error in system configuration updater which occurred after appliance upgrade when static routes have been configured (#291926)
